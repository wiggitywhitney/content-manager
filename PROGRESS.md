## [Unreleased]

### Added
- (2026-05-01) Episode social posts now include the YouTube thumbnail image when dispatching to Bluesky, Mastodon, and LinkedIn. A new `fetchThumbnail()` utility fetches `maxresdefault.jpg` from YouTube's image CDN, falling back to `hqdefault.jpg` on 404. The three platform posters were extended to accept an `imageBuffer` option alongside `videoBuffer`: LinkedIn uses the Images API (single-part PUT, no finalize step — distinct from the Videos API multi-part flow), Mastodon uploads via `v2.media.create` synchronously without polling, and Bluesky uses `agent.uploadBlob` with the `app.bsky.embed.images` embed type. Thumbnail fetch failures are non-fatal: the post goes out as text-only with a warning logged. The `short` post type path is unchanged. 40 new unit tests added; 316 total passing.
- (2026-04-27) Added integration and end-to-end test tiers that run in CI against real credentials. Integration tests (7 tests) verify the Social Posts Queue tab can be read from the staged spreadsheet, that the live production spreadsheet's career-guard column is readable, and that the YouTube Data API returns a valid view count for a known public video. End-to-end tests (3 tests) run the full social post dispatch pipeline in dry-run mode: insert a real row into the Social Posts Queue, spawn `post-social-content.js` as a child process with `DRY_RUN=true`, assert the career guard was checked and dispatch was logged, then delete the test row. Both test tiers run as separate jobs in `daily-sync.yml`. Added `DRY_RUN=true` support to `post-social-content.js`: `dispatchPost()` exits early and logs what it would send; `scanAndPostShorts()` accepts a `dryRun` flag and logs instead of downloading or posting. Unit test count grew from 276 to 281.
- (2026-04-27) Verified that `dispatchPost()` handles `talk` post type rows correctly without any code changes. Talk rows share the same dispatch path as episode rows: no video download, thumbnail fetched from the YouTube URL in Column E, and the Micropub post created with a photo attachment. Added 5 tests to `post-social-content.test.js` covering talk dispatch across all platforms, and a new `post-microblog.test.js` (12 tests) covering `postToMicroblog()` behavior for both talk and episode post types. 276 tests passing.
- (2026-04-27) Completed integration verification for the video embedding feature: confirmed `downloadShortVideo` is defined only in `src/video-download.js` with no duplication, and verified the full test suite passes cleanly.
- (2026-04-27) LinkedIn short posts now embed video inline instead of posting a bare URL. When `postToLinkedIn` receives a video buffer, it runs the full 4-step LinkedIn Videos API flow: initialize upload (get pre-signed URLs and video URN), PUT each 4MB chunk to its upload URL (collecting ETags with surrounding quotes stripped), finalize upload, then poll until the video status is AVAILABLE before creating the post. The post body includes `content.media.id` with the video URN. Text-only posts are unchanged.
- (2026-04-27) Mastodon short posts now embed video inline instead of posting a bare URL. When `postToMastodon` receives a video buffer, it uploads the MP4 via the v2 media API, polls until the attachment URL is populated (Mastodon processes video asynchronously and returns 202 with a null URL initially), then includes the media ID in the status create call. Text-only posts are unchanged.
- (2026-04-27) Bluesky short posts now embed video inline instead of posting a bare URL. When `postToBluesky` receives a video buffer, it acquires a short-lived scoped token from the Bluesky video service (`video.bsky.app`), uploads the MP4, polls until the blob is ready, and includes the embed with a 9:16 aspect ratio in the post. Text-only posts (no video buffer) are unchanged.
- (2026-04-27) Wired video download into the social post dispatcher so that YouTube Short posts download the video once and pass the buffer to all three platform posters (Bluesky, Mastodon, LinkedIn). Previously short posts were dispatched as text-only with a URL; now `dispatchPost()` in `post-social-content.js` detects `post_type = short`, calls the shared `downloadShortVideo()` utility, and forwards the buffer to each platform. The temp directory is always cleaned up in a `finally` block. If the download fails, the row is marked failed immediately with no partial platform dispatch.
- (2026-04-27) Extracted the yt-dlp video download function from the micro.blog module into a standalone shared utility (`src/video-download.js`). Previously `downloadShortVideo()` was a private function inside `post-microblog.js`; it now lives in its own module and is imported back by micro.blog. This is the shared foundation used by the Bluesky, Mastodon, and LinkedIn platform posters — all three import from this module rather than duplicating the yt-dlp logic.
- (2026-04-27) Extended the content scanner to cover three additional shows: Datadog Illuminated, Enlightning, and You Choose. Previously only Thunder and Software Defined Interviews were auto-detected; episodes from the other shows had to be added to the spreadsheet by hand. The scanner now fetches all five YouTube playlists in parallel and deduplicates against existing spreadsheet URLs. The daily GitHub Actions workflow now runs the scan automatically before the post-priority step, so the spreadsheet stays current without manual intervention. 19 unit tests cover playlist ID usage, show name labels, private video filtering, URL format, and pagination. 213 tests passing total.
- (2026-04-25) Launched the dynamic About page: `updateAboutPage()` reads the live production spreadsheet, determines which content channels are currently active based on freshness thresholds, and pushes updated content to the Micro.blog About page via XML-RPC — only when the content has actually changed. The page now shows a photo+bio header (HTML flex layout) followed by a flat list of active channel links. Channels drop off automatically when they go quiet; SDI is always last. Discovered and fixed three Micro.blog API quirks during live testing: `getPages` requires the username as blogId (not integer 1), the page identifier field is `id` not `pageID`, and integers in responses use `<i4>` not `<int>`. 190 tests passing.
- (2026-04-25) Added channel config and activity detection for the dynamic About page: `src/config/about-page-channels.js` defines which content channels (Thunder, Enlightning, Datadog Illuminated, You Choose, Conference Talks, GitHub, SDI) appear based on freshness thresholds, and `getActiveChannels()` in `src/update-about-page.js` filters spreadsheet rows by type and show name to determine which channels have published recently enough to appear. 22 unit tests cover all activity scenarios. Channels sort by most recent content date; SDI always appears last regardless of activity.
